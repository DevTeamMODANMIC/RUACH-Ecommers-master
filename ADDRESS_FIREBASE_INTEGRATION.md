## 🔄 Automatic Migration

### Migration Flow

When a user logs in, the system automatically:

1. **Checks localStorage** for existing addresses
2. **Checks Firebase** to see if addresses already exist
3. **Migrates** if localStorage has addresses but Firebase doesn't
4. **Cleans up** localStorage after successful migration
5. **Shows notification** to user about migration

### Migration Code

Located in: `src/lib/migrate-addresses-to-firebase.ts`

```typescript
import { migrateAddressesToFirebase } from '../lib/migrate-addresses-to-firebase'

// Automatic migration in Profile.tsx
const result = await migrateAddressesToFirebase(user.uid)

console.log(`Migrated ${result.migratedCount} addresses`)
console.log(`Errors:`, result.errors)
```

### Migration Result
```typescript
{
  success: boolean        // Overall success status
  migratedCount: number   // Number of addresses migrated
  errors: string[]        // Array of error messages (if any)
}
```

## 🔒 Firebase Security Rules

Add to your `firestore.rules`:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // User Addresses Collection
    match /userAddresses/{addressId} {
      // Users can read their own addresses
      allow read: if request.auth != null && 
                     resource.data.userId == request.auth.uid;
      
      // Users can create addresses for themselves
      allow create: if request.auth != null && 
                       request.resource.data.userId == request.auth.uid;
      
      // Users can update their own addresses
      allow update: if request.auth != null && 
                       resource.data.userId == request.auth.uid;
      
      // Users can delete their own addresses (soft delete)
      allow delete: if request.auth != null && 
                       resource.data.userId == request.auth.uid;
    }
  }
}
```

## 📊 Firestore Indexes

**Required indexes for optimal performance:**

### Composite Index 1:
- Collection: `userAddresses`
- Fields:
  - `userId` (Ascending)
  - `deleted` (Ascending)
  - `createdAt` (Descending)

### Single Field Indexes:
- `userId` (Ascending)
- `isDefault` (Ascending)
- `deleted` (Ascending)

**Create indexes via Firebase Console:**
1. Go to Firestore Database
2. Click "Indexes" tab
3. Click "Create Index"
4. Add the fields above

## 🚀 Features

### ✅ Real-time Synchronization
- Changes sync instantly across all devices
- No manual refresh needed
- Live updates when addresses change

### ✅ Automatic Default Management
- When setting an address as default, others automatically unset
- Only one default address at a time
- Enforced at the database level

### ✅ Soft Delete
- Addresses are marked as deleted, not removed
- Can be recovered if needed
- Filtered out from queries automatically

### ✅ Security Logging
- All address operations logged
- Track: add, update, delete events
- Includes timestamps and metadata

### ✅ Validation Support
- `validated` flag for address verification
- Ready for integration with address validation APIs
- Can mark addresses as verified
